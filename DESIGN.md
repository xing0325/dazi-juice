# 打字爽 (dazi-juice) — 设计稿

> 一个中文打字练习网页，强反馈、爽感优先、可导入自己喜欢的文章背诵式练打。

## 自主决策的关键问题

| 维度 | 决策 | 理由 |
|---|---|---|
| 形态 | 单文件 HTML 网页 | 用户原话是"网站"；零构建可 file:// 直接打开 |
| 技术栈 | Vanilla HTML+CSS+JS，零依赖 | 没必要 React，单页面够 |
| 游戏化口味 | osu!/节奏游戏风 | 用户说"强反馈"，连击/屏抖/粒子/评级最爽 |
| 中文输入 | composition events，按已上屏字符对比 | Monkeytype 翻车就在这；不能按 keydown 算 |
| 音效 | Web Audio API 合成 | 不引入素材文件，单文件可分享 |
| 粒子 | DOM + CSS transforms | 50 个以内 DOM 性能够，比 canvas 易调 |
| 存储 | localStorage | 单用户，不需要后端 |

## 核心玩法

1. **选/导入文章** → 粘贴文本到对话框，保存到本地文章库
2. **开打** → 当前要打的字高亮放大；已打字变灰；输入法弹候选词没事，按已上屏比对
3. **每打对一字**：
   - 上屏字闪绿、轻微 pop scale
   - 粒子从该字位置爆出
   - 屏幕极轻抖（5px 内）
   - combo +1，combo 大字飙升+缩放
   - 清脆 tick 音（音高随 combo 升）
4. **打错**：
   - 屏幕红闪 + 大抖
   - 低频蜂鸣
   - combo 归零，归零动画
5. **combo 里程碑**（50/100/200/500）：
   - 满屏速度线
   - 巨大 "SUPER"/"INSANE" 字幕飞过
   - 音效升级
6. **完成文章** → 结算页
   - 大字 CPM（字/分）、准确率、用时、最大 combo
   - 评级 S/A/B/C/D（基于速度+准确率组合）
   - 庆祝粒子雨
   - 「再来一次」「换文章」「分享」按钮

## 评级规则（首版）

- CPM = 已上屏字数 / 用时分钟数
- 准确率 = 正确字数 / 总输入字数
- 综合分 = CPM × 准确率
- S: ≥150  /  A: 100-150  /  B: 60-100  /  C: 30-60  /  D: <30

## YAGNI 砍掉的功能

- ❌ 多人对战 / 排行榜（自己练，不需要）
- ❌ 账号系统（localStorage 够了）
- ❌ 教学课程（不是新手向）
- ❌ 五笔/双拼支持（用户没说，默认拼音）
- ❌ PWA/离线（单 HTML 本来就离线）
- ❌ 移动端优化（v1 桌面优先，能用就行）
